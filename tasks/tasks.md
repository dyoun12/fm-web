# 프로젝트 태스크 개요

## 진행 원칙
- 전체 일정은 2주 단위 스프린트를 기준으로 계획하고, 완료된 산출물은 스프린트 리뷰에서 검증한다.
- 모든 태스크는 완료 시 `/docs/` 내 관련 문서와 레퍼런스를 동기화한다.
- 기능 구현 완료 기준은 테스트(단위, 통합, 접근성)와 보안 점검까지 포함한다.
