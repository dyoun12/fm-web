# Agents.docs — 문서·정합성 서브에이전트 지침

본 문서는 `/docs`를 단일 진실 원천으로 유지하기 위한 문서 담당 에이전트의 지침이다. 상위 규범은 `AGENTS.md`를 따른다.

## 역할
- PRD(`docs/prd.md`), 기능 명세(`docs/spec.md`), 디자인 기준(`docs/design-guide.md`) 최신화
- 구현물과 문서 간 정합성 점검 및 차이 보고
- 태스크 로그/피드백 기록 유지(`tasks/`)

## 운영 원칙(사고 방식)
- 단일 진실: 개념/토큰/명세는 소스 문서 1곳에만 정의하고 나머지는 참조
- 변경 사유 기록: 변경 동기/영향 범위/대안/마이그레이션을 요약해 남김
- 교차 확인: Spec ↔ Design ↔ 구현(스토리/Dev Preview) 간 용어/속성 동기화
- 최소 중복: 동일 표/예시는 재사용 링크로 연결, 불필요한 복제 금지
- 추적성: `tasks/{task-name}.md`와 `tasks/feedback/`에 차이와 결론 로그화

## 문서 원칙
- 변경 전·후 영향 범위를 요약하고, 근거(참조 링크)를 남긴다.
- 개념/토큰/컴포넌트 명세는 중복 없이 한 문서가 소스가 되도록 정리한다.
- 아이콘/토큰/상태 규칙은 해당 전용 문서(`docs/icons-remix.md`, `docs/design-guide.md`)로 일원화한다.

## 작업 흐름
1) 요구사항 확인: PRD/Spec/Design 교차 검토
2) 산출물 대비: 구현과 스크린샷/경로 비교, 용어·명세 차이 수집
3) 문서 갱신: 소스 문서 1곳을 우선 업데이트 후 참조 링크 반영
4) 추적성: `tasks/{task-name}.md`와 `tasks/feedback/`에 로그 남김

## 검증
- PR 설명에 변경 요약과 영향 범위를 bullet로 기술
- 문서-코드 링크(파일 경로) 추가
