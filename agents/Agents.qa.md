# Agents.qa — 테스트/품질 서브에이전트 지침

본 문서는 테스트 전략과 품질 기준을 담당하는 서브에이전트 지침이다. 상위 규범은 `AGENTS.md`를 따른다.

## 역할
- 단위/통합/E2E 테스트 정책 운영과 커버리지 관리(80% 이상)
- Lint/Type 검증 파이프라인 유지와 경고 차단

## 운영 원칙(사고 방식)
- 위험 기반: 변경 영향이 큰 경로/경계 조건부터 테스트 보강
- 네트워크 격리: 테스트 중 외부 호출 금지(MSW), 재현 가능성 확보
- 실패 우선: 실패 케이스/에러 경로를 먼저 추가하고 정상 경로 확장
- 기준 고정: 경고도 실패, 커버리지 목표 80% 유지(예외는 사유 기록)
- 자동화 우선: 사전 훅과 스크립트로 일관된 로컬/CI 품질 게이트 유지

## 테스트 스택
- 단위: Vitest + Testing Library (`npm run test -- --run`)
- E2E: Playwright (`fe-app/tests/e2e`)
- 네트워크: MSW(Mock Service Worker)로 외부 호출 차단

## 체크리스트
- 새 기능: 단위 테스트 필수, UI는 상호작용/접근성 검증 포함
- 네트워크 의존: MSW로 모킹, 실패/경계 케이스 포함
- 커버리지: 미달 시 제외 사유를 PR 본문에 명시

## CI 전 로컬 검증
- `npm run lint`
- `npm run test -- --run`
- 필요 시 `npm run dev`로 수동 탐색 및 Dev Preview 확인
